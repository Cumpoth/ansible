2.
Создание пользователей
Необходимо создать плейбук, который создает пользователей user1, user2 с группами smpt,
docker, sudo на удаленном сервере.

В файле ansible/roles/ansible_exercise/defaults/main.yml создаем переменные:
three_groups:
  - smpt
  - sudo
  - docker
В файле ansible/roles/ansible_exercise/tasks/main.yml пишем задачи для ансибл.
Создаем группы на удаленной машине:

- name: Create group smpt
  ansible.builtin.group:
    name: smpt
    state: present

- name: Create group docker
  ansible.builtin.group:
    name: docker
    state: present

- name: Create group sudo
  ansible.builtin.group:
    name: sudo
    state: present

И вносим пользователей в эти группы:

- name: Create user1 with groups
  ansible.builtin.user:
    name: user1
    shell: /bin/bash
    group: "{{ item }}"
    append: yes
  loop: "{{ three_groups }}"

- name: Create user2 with groups
  ansible.builtin.user:
    name: user2
    shell: /bin/bash
    group: "{{ item  }}"
    append: yes
  loop: "{{ three_groups }}"


Значит для пользователей user1, user2 этот плейбук должен сгенерировать ключи формата
ecdsa 521 bit, публичные ключи зарегистрировать на удаленном сервере, приватные ключи
положить локально на хосте, где будет происходить запуск ansible playbook.
Приватные ключи должны быть сгенерированы с паролем - который тоже должен
генерироваться во время выполнения плейбука.
Требования к паролю - длина 20 символов из набора A-Za-z0-9.
Пользователю должны вывестись эти сгенерированные пароли либо в stdout во время
выполнения плейбука, либо в какой либо текстовый файл на хосте где запускается плейбук.


- name: Password ssh key for user1
  ansible.builtin.shell:
    cmd: pwgen -1 -n 20 > /home/xoth/ansible/password_for_user1.txt
  args:
  delegate_to: localhost

- name: Password ssh key for user2
  ansible.builtin.shell:
    cmd: pwgen -1 -n 20 > /home/xoth/ansible/password_for_user2.txt
  args:
  delegate_to: localhost





- name: Create a 521-bit SSH key for user1
  ansible.builtin.user:
    name: user1
    generate_ssh_key: yes
    ssh_key_passphrase: /home/xoth/ansible/password_for_user1.txt
    ssh_key_bits: 521
    ssh_key_file: .ssh/id_rsa
    ssh_key_type: ecdsa

- name: Create a 521-bit SSH key for user2
  ansible.builtin.user:
    name: user2
    generate_ssh_key: yes
    ssh_key_passphrase: /home/xoth/ansible/password_for_user2.txt
    ssh_key_bits: 521
    ssh_key_file: .ssh/id_rsa
    ssh_key_type: ecdsa





- name: display password user1 in stdout 
  ansible.builtin.shell: 
    cmd: cat /home/xoth/ansible/password_for_user1.txt
  register: password_user1
  args:
  delegate_to: localhost
- debug:
    msg: "{{ password_user1.stdout_lines }}"

- name: display password user2 in stdout
  ansible.builtin.shell:
    cmd: cat /home/xoth/ansible/password_for_user2.txt
  register: password_user2
  args: 
  delegate_to: localhost
- debug:
    msg: "{{ password_user2.stdout_lines }}"


- name: set a passphrase for the user1 ssh key
  ansible.builtin.user:
    name: password_user1
    ssh_key_passphrase: "{{ password_user1.stdout_lines }}"

- name: set a passphrase for the user2 ssh key
  ansible.builtin.user:
    name: password_user2
    ssh_key_passphrase: "{{ password_user2.stdout_lines }}"


- name: create file for password for user1
  file: 
    path: "/home/user1/password_for_user1.txt"
    state: touch
  become: yes   

- name: create file for password for user2
  file: 
    path: "/home/user2/password_for_user2.txt"
    state: touch
  become: yes   
        

- name: password transfer for user1
  ansible.builtin.copy:
    src: /home/xoth/ansible/password_for_user1.txt
    dest: /home/user1/password_for_user1.txt

- name: password transfer for user2
  ansible.builtin.copy:
    src: /home/xoth/ansible/password_for_user2.txt
    dest: /home/user2/password_for_user2.txt


- name: set
  ansible.builtin.fetch:
    src: /home/user1/.ssh/id_rsa
    dest: /home/xoth/ansible

- name: set
  ansible.builtin.fetch:
    src: /home/user2/.ssh/id_rsa
    dest: /home/xoth/ansible


Установка docker, docker-compose
Создать плейбук с помощью которого можно будет проводить установку docker+dockercompose последней версии на удаленный сервер.

- name: add GPG key
  apt_key:
    url: http://download.docker.com/linux/ubuntu/gpg
    state: present

- name: add docker repository to apt
  apt_repository:
    repo: deb http://download.docker.com/linux/ubuntu bionic stable
    state: present


- name: install docker
  apt: 
    name: "{{item}}"
    state: latest
    update_cache: yes
  loop: 
    - docker-ce
    - docker-ce-cli
    - containerd.io

- name: make sure that docker was started and enabled
  service:
    name: docker
    state: started
    enabled: yes

Развертывание squid
Нужно чтобы при запуске ansible плейбука он брал шаблонные файлы (docker-compose.yml,
squid.conf) и можно было задать параметр squid_user, который бы добавлялся в users.txt
вместе с паролем.

- name: Template a file to /home/xoth/ansible
  ansible.builtin.template:
    src: "{{ template_docker }}"
    dest: /home/user1
    owner: user1  

- name: Template a file to /home/xoth/ansible
  ansible.builtin.template:
    src: "{{ template_squid }}"
    dest: /home/user1
    owner: user1 

 
- name: Template a file to /home/xoth/ansible
  ansible.builtin.template:
    src: "{{ template_docker }}"
    dest: /home/user2
    owner: user2 

- name: Template a file to /home/xoth/ansible
  ansible.builtin.template:
    src: "{{ template_squid }}"
    dest: /home/user2
    owner: user2

Идеальный - пароль генерится, шифруется и хеш попадает в user.txt, сам пароль
выводится как результат команды:

- name: Password for users

  args:
  delegate_to: localhost
  loop: "{{ squidpass_user }}"
